# provider-registry

## Purpose

Подключаемая система адаптеров, чтобы одна программа могла показывать usage для нескольких провайдеров без хардкода списка в main-цикле.

## Requirements

### Requirement: Интерфейс адаптера

Система MUST определить интерфейс `Adapter` как минимум с:

- `Host() string` — hostname, который обслуживает адаптер (например, `"api.z.ai"`, `"opencode.ai"`).
- `Fetch(ctx context.Context, apiKey string) (Limits, error)` — получив API-ключ из CCR, сходить за usage и вернуть срез `Limit`.
- `NeedsSessionCreds() bool` — true, если адаптер читает дополнительные учётные данные из env или файла конфига до вызова `Fetch`.

`Limit` MUST быть `{ Label string; UsedPct float64; ResetAt *time.Time; Detail string }`. `UsedPct` в замкнутом диапазоне `[0, 100]`.

#### Scenario: Адаптер совпадает со своим хостом

- **WHEN** `Host` провайдера равен `Host()` адаптера
- **THEN** реестр использует этот адаптер для получения usage по этому провайдеру

#### Scenario: Два провайдера с одним хостом

- **WHEN** в реестре два провайдера с одним хостом (например, потому что у них разные ключи)
- **THEN** используется один и тот же экземпляр адаптера; каждый вызов `Fetch` получает соответствующий ключ

### Requirement: Обработка неизвестного хоста

Система MUST выводить провайдера, даже если ни один адаптер не совпал с его хостом. Рендерер MUST выводить `skip (no adapter)` для такой строки.

#### Scenario: Хоста нет в реестре

- **WHEN** у провайдера нет зарегистрированного адаптера
- **THEN** система всё равно выводит его со строкой `skip (no adapter)` и однострочным пояснением

### Requirement: Изоляция ошибок

Сбой (сетевая ошибка, ошибка авторизации, битый ответ) в одном адаптере MUST NOT мешать остальным провайдерам. Система MUST собирать ошибки по адаптерам и рендерить `skip (причина)` для упавших строк.

#### Scenario: Адаптер уходит в таймаут

- **WHEN** HTTP-вызов адаптера уходит в таймаут через 10 секунд
- **THEN** остальные провайдеры всё равно опрашиваются, а упавшая строка показывает `skip (timeout)` или эквивалент

#### Scenario: Адаптер вернул ошибку

- **WHEN** `Fetch` адаптера вернул ненулевую ошибку
- **THEN** система запоминает сообщение ошибки и продолжает с другими провайдерами

### Requirement: Параллельный опрос

Система MUST запускать вызовы `Fetch` параллельно, не более 4 одновременно. Каждый вызов MUST отменяться через корневой контекст программы.

#### Scenario: Медленный провайдер

- **WHEN** один провайдер отвечает 9 секунд, а другой 1 секунду
- **THEN** результат второго появляется в выводе примерно через 1 секунду, а не через 9
