# cli-report

## Purpose

Отрисовать собранные строки провайдеров как простой текст, одна строка на лимит, со строками `skip` для неизвестных хостов и упавших адаптеров.

## ADDED Requirements

### Requirement: Формат вывода

Система MUST писать plain-text отчёт в stdout. Для каждого уникального провайдера, в порядке конфига:

- Заголовочная строка вида `<name-or-host> <host>` (например, `zai api.z.ai`). Если имя тарифа известно (например, `Pro` для z.ai, `Go` для OpenCode), оно дописывается в конец заголовочной строки через пробел.
- Если у провайдера нет адаптера, строка MUST быть `<host> skip (no adapter)` одной строкой.
- Если адаптер вернул sentinel-ошибку, строка MUST быть `<host> skip (<сообщение ошибки>)` одной строкой.
- Иначе по одной строке на `Limit`. Строка начинается с двух пробелов, затем label, выровненный по 14 символов, затем процент использования, затем опциональные поля `Detail` (например, `remaining 82`) и `resets <время>`. Части `Detail` и `resets` опускаются, если неприменимы.

Между блоками провайдеров MUST печататься пустая строка.

#### Scenario: Z.ai с двумя типами лимитов

- **WHEN** адаптер z.ai вернул `Limits{TIME_LIMIT{0%, remaining 100, resetAt 2026-04-16 10:31 SGT}, TOKENS_LIMIT{18%, resetAt +3h}}`
- **THEN** вывод содержит:

  ```text
  zai api.z.ai Pro
    TIME_LIMIT       0%  remaining 100  resets 2026-04-16 10:31 SGT
    TOKENS_LIMIT    18%                 resets in 3h
  ```

#### Scenario: OpenCode Go со всеми тремя окнами

- **WHEN** адаптер OpenCode Go вернул `Limits{rolling(40%, +3h42m), weekly(31%, +4d12h), monthly(21%, +16d5h)}`
- **THEN** вывод содержит:

  ```text
  opencode-go opencode.ai Go
    rolling (5h)   40%  resets in 3h 42m
    weekly         31%  resets in 4d 12h
    monthly        21%  resets in 16d 5h
  ```

#### Scenario: Неизвестный хост

- **WHEN** у провайдера нет зарегистрированного адаптера
- **THEN** вывод содержит одну строку `<host> skip (no adapter)`

#### Scenario: Ошибка адаптера

- **WHEN** адаптер вернул ошибку
- **THEN** вывод содержит одну строку `<host> skip (<сообщение ошибки>)`; сообщение ошибки — точная строка, возвращённая адаптером, с обрезанными крайними пробелами

### Requirement: Порядок и стабильность

Вывод MUST перечислять провайдеров в порядке их первого появления в конфиге CCR (после дедупа). Два последовательных запуска на одном конфиге MUST давать побайтово идентичный вывод, за исключением зависящего от времени выполнения форматирования времени сброса.

#### Scenario: Стабильный порядок

- **WHEN** в конфиге провайдеры `zai`, `opencode`, `yadro`, `yadev`
- **THEN** вывод перечисляет их в этом порядке, независимо от того, сколько занял каждый запрос

### Requirement: Отображение времени сброса

`Limit.ResetAt` MUST рендериться как относительное время («in 3h 42m», «in 4d 12h», «in 16d 5h»), если цель в будущем и в пределах 30 дней, и как абсолютное локальное время («2026-04-16 10:31») в противном случае. Используется локальная таймзона пользователя.

#### Scenario: Близкий сброс

- **WHEN** `ResetAt` — через 3 часа 42 минуты от сейчас
- **THEN** вывод показывает `resets in 3h 42m`

#### Scenario: Далёкий сброс

- **WHEN** `ResetAt` — более чем через 30 дней от сейчас
- **THEN** вывод показывает `resets 2026-08-15 14:00` в локальном времени

#### Scenario: Прошедший сброс

- **WHEN** `ResetAt` в прошлом
- **THEN** вывод показывает `resets now` или опускает часть про сброс

### Requirement: Ненулевой код выхода при инфраструктурной ошибке

Программа MUST завершаться со статусом 0, если хотя бы один провайдер был успешно опрошен ИЛИ в конфиге нет провайдеров. Она MUST завершаться ненулевым кодом только если сам файл конфига отсутствует или некорректен. Сбои отдельных адаптеров НЕ влияют на код выхода.

#### Scenario: Все адаптеры упали

- **WHEN** каждый адаптер вернул ошибку, но конфиг валиден
- **THEN** программа завершается с кодом 0 и печатает соответствующие строки `skip`

#### Scenario: Конфиг отсутствует

- **WHEN** файла конфига не существует
- **THEN** программа завершается с кодом 2 и печатает однострочное сообщение об ошибке в stderr
